# 深度信息提取方法

## 支持信息

- 商户基础信息：名称、地址、电话、营业时间、商圈、交通。
- 评分信息：综合评分、口味、环境、服务、食材。
- 用户标签：标签名、提及次数、情感倾向和类别。
- 推荐菜：菜名、价格、图片信息。
- 评论内容：摘要、发布时间、用户信息、图片。
- 商圈信息：整体评分、热门标签、地理范围、交通便利性、热门店铺。

## 首选方案：readability

```javascript
use_browser(action="readability")
```

优点：内容干净、信息完整、适合详情页和商圈页。

## 备用方案：snapshot

```javascript
use_browser(action="snapshot", format="ai")
```

用于精确定位评分、标签或推荐菜等特殊元素。

## 标准化格式

```json
{
  "overall_score": 4.8,
  "category_scores": {
    "taste": 4.7,
    "environment": 4.6,
    "service": 4.5,
    "ingredients": 4.7
  },
  "comment_tags": {
    "适合逛吃": 3641,
    "停车建议": 81
  }
}
```

## 异常处理

- 字段缺失：标记为“暂无信息”。
- 格式异常：提取数字部分并保留原始文本。
- 加载不完整：等待后重试一次。
- 验证码/滑块：暂停并让用户手动处理。
