# Travel Skill Reference

## 定位

`travel-skill` 是城市旅游场景的基础推荐和工具增强 skill。它负责把用户的城市、时长、预算、人流偏好、同行类型和补充线索，整理成可执行的建议。

## 已支持

- 城市维度景点推荐
- 城市维度美食推荐
- 玩 + 吃组合推荐
- 基于人流偏好的差异化输出
- 基于时长、预算、出行类型的轻量适配
- 利用用户上传图片或文字中的显式信息进行修正
- 天气查询与出游建议摘要
- 路线规划：步行 / 驾车 / 公交
- 联网检索最新开放时间、门票、预约政策等信息
- POI 卡片化输出，包含详情链接、图片封面、大众点评入口、小红书介绍入口、图片参考入口和高德导航入口
- 飞书组件化推荐卡片字段，支持非 Markdown interactive card 优先输出

## 暂不支持

- 实时精确人流计算
- 实时排队分钟数
- 动态票价监控
- 多城市联游自动拆解
- 酒店与交通一体化预订
- 复杂地图可视化导航

## 请求路由

| 请求类型 | 说明 | 典型问题 |
|----------|------|----------|
| `recommendation` | 常规旅游推荐 | 杭州半天怎么玩？ |
| `weather` | 天气与出游建议 | 上海这两天天气怎么样？ |
| `route` | 起终点路线规划 | 从橘子洲到岳麓山怎么走？ |
| `web_search` | 最新信息联网检索 | 上海博物馆最新开放时间是什么？ |

## 人流等级

| 等级 | 含义 | 用户理解 |
|------|------|----------|
| `S` | 超级热门 | 大概率人多，需要排队或错峰 |
| `A` | 热门 | 常规热门点，周末和节假日较拥挤 |
| `B` | 中等 | 人气不错，多数时间可接受 |
| `C` | 小众 | 相对安静，适合避开主流人流 |
| `D` | 冷门 | 非常安静，适合明确追求人少的用户 |

这些等级是经验风险，不是实时客流。

## POI 富媒体字段

| 字段 | 说明 |
|------|------|
| `detail_url` | 首选详情链接；优先真实详情页，没有时使用大众点评或高德搜索入口，不编造 URL |
| `dianping_url` | 大众点评商户详情页或搜索入口，用于评分、人均、评论标签、排队和本地生活参考 |
| `xhs_url` | 小红书代表笔记或搜索入口，用于近期体验、氛围、拍照和排队体感等软信号 |
| `xhs_intro_url` | 兼容字段，小红书搜索或代表笔记链接；新输出同时提供 `xhs_url` |
| `image_url` | 图片示例入口，优先真实图片页，否则使用图片搜索入口 |
| `image_search_url` | 图片搜索入口，用作图片参考，不声明版权或官方来源 |
| `cover_image_url` | 卡片封面，默认与 `image_url` 或 `image_search_url` 保持一致 |
| `amap_navigation_url` | 高德导航链接；缺少坐标时回退为高德搜索链接 |
| `source_note` | 链接来源、搜索入口和可信度说明 |
| `card` | 单个 POI 的用户展示卡片数据，包含标题、封面、地址、详情链接和操作按钮 |
| `feishu_card` | 可选的飞书 schema 2.0 interactive card；飞书渠道优先使用非 Markdown 组件化展示 |

向用户推荐任何景点、餐厅、商圈或路线点位时，每个点位单独一个卡片，并通过 `cover_image_url` 或“图片示例”按钮提供图片入口。大众点评、小红书和图片字段只作为参考入口，开放时间、票务、官方地址等硬事实仍需以官方或高德等结构化来源为准。

## 工具层

- `tools/amap_client.py`：高德地理编码、POI 搜索、附近搜索、高德导航/搜索链接构造、步行/驾车/公交路线。
- `tools/weather_client.py`：当前天气、3 天预报和出游建议。
- `tools/web_search.py`：开放时间、门票、预约政策等联网检索。
- `tools/version_manager.py`：skill 快照与回滚。
